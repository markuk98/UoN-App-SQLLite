    Version: 3.6.0.01 (12/12/17)

    Features added:

    • Chase function to send emails to users and track history. Please see the new tab. Instructions below.

    Changes made:

    • Loan reminder emails are no longer sent.

    Chase function Instructions

    Please use Google Chrome - Firefox doesn't select all information.

    Adding New Ticket.

    Navigate to the ticket.

    Add the request for information to the notes and update ticket. (Example: Please provide your workstation asset tag to I can log in remotely.)

    Select the 'Chase' tab in the UoN App

    Click 'Add new' Button

    The ticket should be added to the list below.

    Send an email.

    Select the ticket from the first box and click 'Send Notification'

    A new mail windows should open with an email pre-drafted. Check the content, adjust if needed and send when ready.

    Back in the UoN App, double click the ticket you just updated in the first window. This should take you to the ticket so you can update it.

    Select notes and paste. A SEANS comment should be added. Click add and then apply.

    Checking tickets

    The system will highlight any tickets in orange that need an update sending. Follow the instruction above.

    Resolving a ticket

    Ticket can be resolved at any stage. Resolve tickets when the user replies or the reach the end of the three stage process. Remember there is 24 hours after stage three is sent before the can be closed.

    Other features

    You can re-open a closed ticket by re-adding it. Follow the 'Adding new Ticket' instructions. You will be able to resend the stage 3 ticket.

    Only the tickets you are in control of are displayed in your window.

    You can take control of someone else's ticket by adding it again. This will load the history of that ticket too. This will enable you to see who sent which reminder and the date it was sent.

    Please bare in mind that it is possible to close and not send the email. Just because it says it was sent, if the user didn't send it, the customer won't receive it

    It should be saved to your drafts.
